# 国际化（i18n）策略

> 本文档定义 Code Assistant 插件的多语言支持策略和字符串资源组织方式。

## 一、语言支持

| 语言       | 资源文件                        | 覆盖范围                         |
|----------|-----------------------------|------------------------------|
| 简体中文（默认） | `messages_zh_CN.properties` | 全部 UI + System Prompt + 工具描述 |
| 英文       | `messages_en.properties`    | 全部 UI + System Prompt + 工具描述 |

默认语言跟随 IDE 语言设置（`Locale.getDefault()`），未匹配到对应资源文件时回退到简体中文。

## 二、资源文件组织

### 目录结构

```
src/main/resources/i18n/
├── messages_zh_CN.properties    # 简体中文（默认回退）
├── messages_en.properties       # 英文
├── system-prompt_zh_CN.md       # 中文 System Prompt 模板
└── system-prompt_en.md          # 英文 System Prompt 模板
```

### messages.properties Key 命名规范

```
<模块>.<组件>.<用途>

示例：
  ui.chat.send_button=发送
  ui.settings.api_key_label=API Key
  agent.tool.read.description=读取项目内指定文件的内容
  agent.error.rate_limited=请求过于频繁，请稍后重试
  completion.stats.shown=显示次数
  git.commit.button=生成提交信息
  mcp.status.running=运行中
  mcp.status.crashed=已崩溃
```

### Key 分类

| 前缀             | 覆盖范围                                 |
|----------------|--------------------------------------|
| `ui.*`         | 所有 UI 文本（按钮、标签、提示、Toast、对话框）         |
| `agent.*`      | Agent 相关（工具描述、错误消息、System Prompt 片段） |
| `completion.*` | 补全模块（统计标签、配置描述）                      |
| `git.*`        | Git Message 模块（按钮文本、Prompt 模板片段）     |
| `mcp.*`        | MCP 模块（状态文本、错误消息）                    |
| `skill.*`      | Skill 系统（UI 标签）                      |
| `session.*`    | 会话管理（标题默认值、操作确认文本）                   |

## 三、代码集成

### ResourceBundle 加载

```kotlin
object I18n {
    private val bundle: ResourceBundle by lazy {
        ResourceBundle.getBundle("i18n/messages", Locale.getDefault())
    }

    fun get(key: String, vararg args: Any): String {
        val template = try {
            bundle.getString(key)
        } catch (_: MissingResourceException) {
            // 回退到中文
            ResourceBundle.getBundle("i18n/messages", Locale.SIMPLIFIED_CHINESE).getString(key)
        }
        return if (args.isEmpty()) template else String.format(template, *args)
    }
}
```

### 使用方式

```kotlin
// 简单文本
button.text = I18n.get("ui.chat.send_button")

// 带参数
Toast.show(I18n.get("agent.tool.read.lines_truncated", totalLines, maxLines))
// → "共 1200 行，已截断到 500 行" / "1200 lines total, truncated to 500"

// 工具描述（注入 System Prompt）
val toolDescription = I18n.get("agent.tool.read.description")
```

### System Prompt 模板

System Prompt 按语言分文件存储，`buildSystemPrompt()` 加载对应语言的 Markdown 模板后做变量替换：

```kotlin
fun buildSystemPrompt(): String {
    val locale = Locale.getDefault()
    val suffix = if (locale.language == "zh") "zh_CN" else "en"
    val template = javaClass.getResourceAsStream("/i18n/system-prompt_$suffix.md")
        ?.bufferedReader()?.readText()
        ?: javaClass.getResourceAsStream("/i18n/system-prompt_zh_CN.md")
            ?.bufferedReader()?.readText()
        ?: error("System prompt template not found")

    return template
        .replace("{projectName}", project.name)
        .replace("{basePath}", project.basePath ?: "unknown")
        .replace("{dateStr}", LocalDate.now().toString())
        .replace("{osName}", System.getProperty("os.name"))
        .replace("{branch}", getGitBranch())
        .replace("{configRef}", buildConfigRef())
        .replace("{toolSection}", ToolRegistry.buildSystemPromptTools())
        .replace("{skillExt}", SkillManager.getSystemPromptExtension())
}
```

## 四、不翻译的内容

| 内容         | 语言   | 原因                       |
|------------|------|--------------------------|
| API 参数/字段名 | 英文   | JSON key 等，协议层面不可翻译      |
| 日志输出       | 简体中文 | 目标用户为中文开发者，方便排查          |
| 配置文件 key   | 英文   | `mcp-config.json` 等，格式约定 |
| 代码标识符      | 英文   | 类名、方法名、变量名               |

## 五、Git Commit Message 的 Prompt 语言

Commit message 生成语言跟随 IDE 语言设置：

- IDE 语言为中文 → 中文 commit message（默认模板为中文）
- IDE 语言为英文 → 英文 commit message（默认模板为英文）
- 用户自定义 prompt 模板优先，不跟随语言自动切换

## 六、工具描述的多语言

`ToolRegistry` 中的工具 `description` 字段根据当前语言加载：

```kotlin
class ToolRegistry {
    fun buildSystemPromptTools(): String {
        return buildString {
            for (tool in listBuiltin()) {
                val desc = I18n.get("agent.tool.${tool.lowercase()}.description")
                appendLine("### $tool — $desc")
            }
        }
    }
}
```

## 七、开发规范

1. **新增 UI 文本时**：先在 `messages_zh_CN.properties` 和 `messages_en.properties` 中分别添加 key
   和翻译
2. **禁止硬编码**：代码中不出现用户可见的中文/英文字符串字面量，统一用 `I18n.get(key)`
3. **Key 命名**：遵循 `<模块>.<组件>.<用途>` 三级命名，全小写，下划线分隔
4. **翻译质量**：英文翻译需准确、自然，不使用机器翻译；技术术语保持英文（如 "API Key"、"Plugin"）
5. **缺失翻译**：`messages_en.properties` 中缺失的 key 在运行时回退到中文 value，编译时不强制校验（通过
   CI 的 `diff_keys.sh` 脚本检查两个 properties 文件的 key 是否一致）
